---
title: Quickstart
description: Install nanny, configure limits, and run your first enforced process in under five minutes.
---

## Install

```bash
cargo install nanny
```

Verify the installation:

```bash
nanny --version
```

## Initialise a config

Run this in the root of your project:

```bash
nanny init
```

This writes a `nanny.toml` with safe defaults:

```toml
[runtime]
mode = "enforcing"

[limits]
steps   = 100
cost    = 1000
timeout = 30000
```

Edit the values to match your agent's requirements.

## Run your agent

```bash
nanny run python agent.py
nanny run ./my-agent
nanny run node runner.js
```

Nanny reads `nanny.toml` from the current directory, spawns the process, and kills it the moment any limit is crossed.

## Use named limits

Define limit sets for different workloads in the same `nanny.toml`:

```toml
[limits]
steps   = 50
cost    = 500
timeout = 15000

[limits.researcher]
steps   = 200
cost    = 5000
timeout = 120000
```

Then activate a named set at runtime:

```bash
nanny run --limits=researcher python agent.py
```

Named sets inherit from `[limits]` and override only the fields you declare.

## Read the event log

Every run emits structured NDJSON to stdout:

```json
{"event":"ExecutionStarted","ts":1711234567000,"limits":{"steps":100,"cost":1000,"timeout":30000},"limits_set":"[limits]","command":"python agent.py"}
{"event":"ExecutionStopped","ts":1711234572000,"reason":"AgentCompleted","steps":0,"cost_spent":0,"elapsed_ms":4823}
```

Pipe it to a file or your log aggregator:

```bash
nanny run python agent.py >> nanny.log
```

Or configure file output directly in `nanny.toml`:

```toml
[observability]
log      = "file"
log_file = "nanny.log"
```
