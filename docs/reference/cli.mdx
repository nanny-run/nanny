## nanny init

Writes a starter `nanny.toml` in the current directory.

```bash
nanny init
```

Fails if `nanny.toml` already exists. To reset, delete it and run again.

---

## nanny run

Runs a command under nanny enforcement.

```bash
nanny run [OPTIONS] <COMMAND> [ARGS...]
```

### Options

| Flag | Type | Description |
|------|------|-------------|
| `--limits=<name>` | string | Activate a named limit set from `nanny.toml`. Inherits from `[limits]`. |
| `--config=<path>` | path | Path to config file. Defaults to `./nanny.toml`. |

### Examples

```bash
# Run with base [limits]
nanny run python agent.py

# Run with a named limit set
nanny run --limits=researcher python agent.py

# Pass flags to the child command
nanny run -- python agent.py --verbose --output results.json

# Use a config file in a non-standard location
nanny run --config=./config/nanny.toml python agent.py
```

### Exit codes

| Code | Meaning |
|------|---------|
| `0` | Process exited cleanly (`AgentCompleted`) |
| `1` | Nanny stopped the process, or an internal error occurred |

### Stderr output

When nanny stops a process, it prints the reason to stderr:

```
nanny: stopped — TimeoutExpired
nanny: stopped — MaxStepsReached
nanny: stopped — BudgetExhausted
```

This is separate from the structured event log, which goes to stdout (or a file).
