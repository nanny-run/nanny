title: Introduction
description: What Nanny is, what it is not, and when to use it.
---

Nanny is a deterministic execution boundary for autonomous agents and long-running processes.

It enforces hard limits — **steps, cost, and time** — and stops execution the moment a limit is crossed.
No soft warnings. No retries. No negotiation.

## What it does

When you run an agent under nanny, you get a guarantee:

- It will not take more steps than you allow.
- It will not spend more than your cost budget.
- It will not run longer than your timeout.

If any limit is breached, nanny kills the process and emits a structured event log explaining exactly why.

## What it is not

Nanny is not intelligent. It does not understand what your agent is doing.
It does not suggest when to stop, summarize results, or retry on failure.

It is a **primitive** — a hard boundary you configure once and trust completely.

## The standard path

Nanny works with any language and any binary. No code changes required.

```bash
nanny run python agent.py
nanny run ./my-agent
nanny run node runner.js
```

For per-function governance (marking individual tools and rules), the **Rust macros** (v0.2.0)
and **Python SDK** (v0.3.0) extend the same boundary into your code — with identical interfaces
across both languages.

## Version roadmap

| Version | What ships |
|---------|-----------|
| **v0.1.0** | Runtime — `nanny run`, hard process-level limits, structured event log |
| **v0.2.0** | Rust macros — `#[nanny::tool]`, `#[nanny::rule]`, `#[nanny::agent]` |
| **v0.3.0** | Python SDK — `@tool`, `@rule`, `@agent` (public launch) |
| **v0.4.0** | Cloud — team dashboards, audit trails, managed enforcement |
